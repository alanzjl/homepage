Skeleton Tracking

With Kinect v1.0 and openni_tracker, we achieved 4 DOFs skeleton tracking in each arm.

Openni provides 4 transformations for each arm, 3 for shoulder joint and 1 for elbow joint. Our project first extracts each joint's orientation and transformation from this information.





DUAL ARM MAPPING

Use two arms to control baxter arms separately, but with reduced DOF.

Since openni only provides 4 DOFs for each human arm, while one baxter arm has 7 DOFs, we can only control a part of Baxter joints in this mode. We have complete information for the shoulder joint (3 DOFs) and incomplete imformation for the elbow joint (1 DOF). These 4 DOFs act exactly according to filtered reading from Kinect. Wrist joints are set to default position in this mode.





SINGLE ARM MAPPING

Use two arms to control one baxter arm. This control is complete.

Use human left arm's shoulder and elbow to control baxter right arm's shoulder and elbow joints; use human right arm's shoulder to control baxter's wrist; use human right arm's elbow to control baxter's gripper. We can use this mode to achieve more accurate manipulation (e.g. object picking).





FACE DETECTION

Detect and track user's face, do some transformation, add some funny snippets, display on Baxter's face screen.

The implementation for this function can be separated into two parts: face detection and image synthetization. The face detection function is achieved by the OpenCV library. By using the CascadeClassifier() function, we can determine the location of user’s face in the image captured by the camera. After we get the face’s location in the image, we can slice the whole image into face image.





FACE MANIPULATION

Detect and track user's face, do some transformation, add some funny snippets, display on Baxter's face screen.

After getting the image of the user’s face, we can now synthesiz. First, we find a background image and remove Tony Stark’s face. But we cannot simply add the face image on this background. We have to adjust the face image into a resolution of 450x450. But we still cannot put the resized face image on the background, because as we can see on the edge of Tony Stark’s face, there is some gradient change effect. Thus, we the following mask to create such effect. By simply dot product the face image and the mask, we can add some gradient change effect on the user’s face. After adding this effect, we then put the image on the middle of the background. This is the final output.





MOTION RECORDING

We add a playback module which can start/stop recording and start/stop playing on demand.

We add a ROS node which listens to the command from our controller. After receiving a message, add a timestamp to each command, append in a python list, then save to files using python pickle.
