Implementation

Control with Ubuntu, driven by Arduino. Based on OpenCV and ROS.

Control System

our control system is formed with the input of target position and target size and output of the action of our robot. At the very beginning, the robot will rotate clockwise in order to find the target. Once the camera capture the target, it will define the targetâ€™s position and size, conveying the data to CPU and it will drive the robot moving forward and modifying its direction, until the target is put at the ideal location and zoomed into the predefined size on computer. Then the motor will automatically stop, with the launching barrel rotating to find the ejection angle. During this process, the computer continues judging whether the target is static. Once the result of the judgement is True, the drum will rotate to transform the ball into the barrel, friction pulley popping it out with high speed.





Control Units

The control system of this project was built on ROS Indigo, a communication platform hosting on Ubuntu 14.04. Each computing or control unit in ROS is called a node. There are 4 nodes in this project in total.

Image Collecting Node: This node collects, calibrates, and publishes image from camera.
Detector: This node runs our visual algorithm. It subscribes to the image, detects the marker pose and publishes the pose of desired marker.
Controller: This node subscribes to the marker pose topic, listens to keyboard events, and decide the motion of the robot. It published command.
Driver: This node receives command and carries it out by controlling each actuator.





Vision

We use color thresholds to detect markers. Convert the RGB image to HSV image, which has a better performance in color splitting. Apply pre-defined thresholds to each channel. Thresholds can be automatically adjusted when clicking. Apply morphology operators (erosion and dilation) to the mask to isolate individual elements and join disparate elements. Detect contours in the mask and pick the largest one as the target. The distance can be calculated by the ratio between its nominal area in the image and real area.





Motion

Front wheel and rear wheel are coupled, i.e. only left wheel and right wheel are controllable. Robot mobile base will follow the marker if the distance between the marker and the robot is within some threshold. If the distance is smaller than the threshold, mobile base will stop moving, and the fort (z-axis rotation) and the barrel (x-axis, pitch) start moving. When this is accomplished, shooting motor starts working and ball feeder feeds one ping-pong ball to the barrel. If no target is detected, the robot will spin to wait or find one.





Chasis Design

We design the chassis with two separated parts which we use cropper cylinders to brace. The space created by the cropper cylinder is used for placing control board and battery. The number of the cropper per area is based on the load of the area. The heavier the load, the more cropper cylinder. We place four motors on the edges of the lower part, using couplers to connect it with wheels.






Launching Platform

The motor on the left side is the one driving the friction pulley with a high speed. Instead of using one friction pulley, we use two to guarantee the fully contact between friction pulleys and the ball, as well as the avoidance of the vibration of the tennis ball during launching process.
The middle part is our launching barrel. We separate it into four part so that it will save the cost of 3-D printing. Connected by the laser printed bar with servo motor, the launching barrel could rotate along the horizontal axis, which would enable it to adjust the launching angle.
The right part is the tennis ball container and the ball feeder driven by a low speed motor. The ball feeder is designed to be hollow so that it is cost-saving while implement its function perfectly.
